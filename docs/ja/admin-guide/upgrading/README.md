# GROWI のアップグレードと v7 系への移行

GROWI は、最新の v7 系（執筆時点の最新は v7.5.x）を利用することを推奨します。
最新系列では機能・パフォーマンス・セキュリティの改善が継続的に行われており、サポート対象の実行環境も最新化されています。
このページは、古いバージョンの GROWI を運用している場合に「どの経路で」「どこで一度立ち止まって作業すべきか」「どのバージョンで途中下車してはいけないか」を俯瞰するための案内です。

各マイナーバージョンの具体的な変更点・手順は、個別の「GROWI vX.Y.x へのアップグレード」ページにまとまっています。
アップグレードを実施する前に、通過するすべての系列の個別ガイドを必ず確認してください。

::: tip
最低でも v7.4.0 以降を利用することを推奨します。
後述の Revision データマイグレーションに関する安全策が、v7.4.0 以降では自動で適用されます。
:::

## アップグレードの基本的な考え方

GROWI は起動時にデータベースのマイグレーションを自動的に実行するため、原則として複数のバージョンを飛ばして一度にアップグレードできます。
ただし、次の 2 点は例外的に注意が必要です。

- **一度立ち止まって手作業を行うべきポイント** がある（特に v4.x 以前からの「v5 互換形式」への変換）
- **途中下車（起動・常用）してはいけないバージョン範囲** がある（Revision データマイグレーションバグ）

以降でそれぞれを説明します。


## 現在のバージョン別・推奨経路

現在運用しているバージョンごとの、推奨するアップグレード経路は次のとおりです。

| 現在のバージョン | 推奨経路 | 特に注意すべき作業 |
| :--- | :--- | :--- |
| v4.x 以前 | v5 系 → v6 系 → v7.4 以降 | **v5 互換形式への変換**（別格の大作業。下記参照） |
| v5.x / v6.0.x | v7.4 以降へ | 実行環境（Node.js / MongoDB / Elasticsearch）の要件確認 |
| v6.1.0〜v7.0.15 | できるだけ早く v7.4 以降へ | **この範囲での運用継続を避ける**（下記参照） |
| v7.0.16 以降 | 最新系列へ | 個別ガイドの変更点確認 |

::: warning
v6.0.15 以前から v7 系へ上げる場合は、v6.1.0〜v7.0.15 のバージョンで起動・常用したまま留まらないでください。
理由は [避けるべき「途中下車」バージョン](#避けるべき「途中下車」バージョン) を参照してください。
:::


## 一度立ち止まるべきポイント

### v4.x 以前 → v5：v5 互換形式への変換（最大の作業）

v4.x 以前から v5 系にアップグレードした後の **「v5 互換形式」への変換が、一連のアップグレードの中で最も大きな作業** です。
v5.0 でページの内部データ形式が大きく変わり、v4.5 以前に作成されたページは自動では変換されません。
ページツリーへの表示や、移動・リネーム・削除時の子孫ページの扱いに関わるため、計画的に実施する必要があります。

詳細な手順・影響範囲は [GROWI v5.0.x へのアップグレード](/ja/admin-guide/upgrading/50x.html) を参照してください。

### 実行環境の要件を満たすタイミング

通過する系列ごとに、実行環境の前提が変わります。
アップグレード前に現在の環境を確認してください。

- **Elasticsearch 6 → 7**（v5.0 で対応）— [v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html)
- **Node.js 16 のサポート終了（18 / 20 へ）**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
- **Bootstrap v4 → v5 記法の手動書き換え**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
- **HackMD 連携機能の廃止**（v7.0。ビルトインエディタの同時編集へ移行）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
- **MongoDB 6.0 以上が必須**（v7.1）— [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html)


## 避けるべき「途中下車」バージョン

v6.1.0 から v7.0.15 までのバージョンには、Revision（ページの更新履歴）データのマイグレーションに関する不具合があります。
これらのバージョンで起動・運用すると、本来は別々のページに紐づくべき Revision が単一のページに誤って関連付けられ、非公開ページや削除済みページの内容・作成者情報が意図せず閲覧可能になるおそれがあります。

このため、アップグレードの経路として v6.1.0〜v7.0.15 のバージョンで起動・常用したまま留まらないことを強く推奨します。

| バージョン範囲 | 状態 |
| :--- | :--- |
| v5.0.0〜v6.0.15 | 安全 |
| v6.1.0〜v7.0.15 | **危険（この範囲での運用を避ける）** |
| v7.0.16 以降 | 安全策あり（v7.4.0 以降は自動で適用） |

推奨される安全な経路は次のとおりです。

- v6.0.15 以前から上げる場合は、v6.1.0〜v7.0.15 で途中下車せず **v7.4.0 以降へ直接アップグレード** する
- 新規に構築する場合は、v7.0.16 以降（推奨は最新系列）で構築する

::: danger
この不具合の詳細（影響条件・原因・確認方法・復旧手順）は、開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を必ず確認してください。
すでに該当範囲を経由して運用している場合の対応も、こちらに記載されています。
:::


## アップグレード前後の確認

- アップグレード前に、必ずデータベースおよび添付ファイルのバックアップを取得してください
- 通過するすべての系列の個別ガイドの「アップグレード前にチェックすべきこと」を確認してください
- 大規模な変換作業（v5 互換形式など）を伴う場合は、メンテナンスモードの活用を検討してください
