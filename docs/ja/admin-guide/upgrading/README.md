# GROWI 最新バージョンへの移行ガイド
GROWI の過去バージョンをご利用の皆様に向けて、**GROWI 最新バージョンへのアップデート方法（2026年6月時点では v7.5.x）**を横断的にご案内します。

最新系列では機能・パフォーマンス・セキュリティの改善が継続的に行われており、サポート対象の実行環境も最新化されています。
皆様に安全にGROWIをご利用し続けていただくためにも、開発チームとしては最新バージョンへのアップデートを強く推奨しております。

各マイナーバージョンの具体的な変更点は個別のアップグレードページにも掲載されておりますが、本ページでは最新バージョンへ一気にアップデートされる場合に実施が必要な作業、確認が必要な項目、注意点に絞ってご紹介します。

::: danger
Revision（ページの更新履歴）データの重大な不具合があるため、**v6.1.0〜v7.0.15 にはアップグレードせず、必ず v7.4.0 以降へ直接**アップグレードしてください。
詳細は開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を参照してください。
:::

アップグレード前には、必ず MongoDB のバックアップを取得してください。

## 〜v4.x を利用中の場合
### Elasticsearch を 7.x にアップグレード、インデックス再生成
- Elasticsearch を利用している場合は、まず **Elasticsearch 6 → 7** へのアップグレードが必要です。
- Elastic 6 系と 7 系は公式にはインデックスに互換性があるとされていますが、GROWI での利用にあたってはトラブルを避けるため、既存のインデックスデータの全消去およびインデックスの再生成をおすすめします。
  - [v5.0.x へのアップグレード /Elasticsearch 7.x のサポート](/ja/admin-guide/upgrading/50x.html#elasticsearch-7-x-%E3%81%AE%E3%82%B5%E3%83%9B%E3%82%9A%E3%83%BC%E3%83%88)

### GROWI 本体を v5.0.11 へアップグレード
- [v5.0.x へのアップグレード](/ja/admin-guide/upgrading/50x.html)に沿って、GROWI 本体を**v5.0.11 へアップグレード**してください。

### ページデータを v5 互換形式へ変換
- v5.0.x へのアップグレード後、ページデータを**v5 互換形式へ変換**する作業が必要になります。
- 手順・影響範囲は [v5.0.x へのアップグレード / 新しい v5 互換形式について](/ja/admin-guide/upgrading/50x.html#新しい-v5-互換形式について) を参照してください。

## GROWI 本体以外のアップグレード・設定変更 (v5.x〜 を利用中の場合)

最新バージョンへアップグレードする前に、**実行環境（ミドルウェア）の更新や、環境変数・インフラまわりの設定変更**のうち、お使いのバージョンから最新までに必要なものを済ませておく必要があります。

現在ご利用中のバージョンの項を起点に、**そこから下（より新しいバージョン側）の対応項目をすべて順に確認・実施**してください。
各変更の詳細・背景は、リンク先の個別アップグレードガイドの「管理者向け」セクションを参照してください。

環境変数の一覧は [環境変数](/ja/admin-guide/admin-cookbook/env-vars.html) を参照してください。

### v5.x を利用中の場合
参考：[v6.0.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/60x.html#管理者向け)

ミドルウェアのバージョン要件に変更はありません。一方で、以下の管理画面・認証まわりの設定は自動では引き継がれないため、該当する場合は対応してください。

- [該当時] **XSS(クロスサイトスクリプティング)対策設定**
  - 過去の設定は引き継がれず、起動時にデフォルト（ON・「おすすめ設定」）にリセットされます。
  - 独自にタグ・属性をホワイトリスト登録していた場合は再設定してください（許可属性の記述形式が JSON に変更されています）。
- [該当時] **カスタム HTML Header の廃止**
  - head タグへ任意の文字列・タグを挿入する機能は廃止されました。利用していた場合は、新設された「カスタム Noscript」やカスタムスクリプトへ移行してください。
- [該当時] **Twitter OAuth 2 認証の廃止**
  - 利用していた場合は別の認証方式へ移行してください。
- [該当時] **nocdn 版の廃止**
  - CDN を使わない nocdn 版を常用していた場合、公式イメージは CDN 利用を抑えた 1 本に統合されています。

### v6.0.x を利用中の場合
参考：[v6.1.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/61x.html#管理者向け)

- [必須] **Node.js アップグレード**
  - v18 にアップグレードしてください。
  - 公式 Docker イメージ利用時は対応不要です。
- [必須] **Elasticsearch アップグレード**
  - v8 にアップグレードしてください。
- [該当時] **`FILE_UPLOAD=local`（添付ファイルをローカルファイルシステムに保存）の場合**
  - 保存場所が `/opt/growi/packages/app/public` から `/opt/growi/apps/app/public` に変更されました。
  - アップグレード後に既存ファイルを移動してください。
  - AWS S3・GCP GCS・MongoDB GridFS 利用時は対応不要です。
- [該当時] **ソースから自前でビルドしている場合**
  - ビルドツールが Lerna から Turborepo に変更されました。
  - 公式 Docker イメージ利用時は対応不要です。

### v6.1.x 〜 v6.2.x を利用中の場合
参考：[v6.3.x へのアップグレード](/ja/admin-guide/upgrading/63x.html)

- [該当時] **GROWI v3.3 系以前に構築し、添付ファイルを MongoDB GridFS で管理しているシステム**
  - `/attachment/{pageID}/{fileName}` 形式のエンドポイントが廃止されます。
  - 該当 URL を含むページは `/attachment/{attachmentId}` 形式に書き換えるか、ファイルを再アップロードしてください。

### v6.3.x を利用中の場合
参考：[v7.0.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/70x.html#管理者向け)

- [必須] **Node.js アップグレード**
  - v20 にアップグレードしてください。
  - 公式 Docker イメージ利用時は対応不要です。
- [要確認] **Bootstrap アップグレード**
  - Bootstrap が v4.6 → v5.3 にアップグレードされています。
  - Markdown 中で Bootstrap の記法を使った HTML タグに影響するため、利用者への周知が必要です。
- [該当時] **HackMD 連携の廃止**
  - HackMD と連携して複数人で同時に編集する機能は廃止されました。
  - ビルトインエディタで同時多人数編集が可能になります。
- [該当時] **Promster 連携の廃止**
  - Promster で監視していた場合は監視構成を見直してください。

### v7.0.x を利用中の場合
参考：[v7.1.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/71x.html#管理者向け)

- [必須] **MongoDB アップグレード** 
  - v6.0 以上にアップグレードしてください。
- [該当時] **AWS S3 にファイルを保存している場合**
  - アップロード時にオブジェクト ACL（`public-read`）が付与されなくなりました。
  - AWS のベストプラクティスに従い、S3 バケットの ACL 無効化・パブリックアクセスブロックを推奨します。
  - 環境変数 `S3_OBJECT_ACL=public-read` を設定している場合は `private` に変更するか削除してください（バケット設定を変えず従来どおり運用する場合のみ `public-read` のまま）。
- [該当時] **ソースから自前でビルドしている場合**
  - パッケージマネージャー・タスクランナーが yarn (v1) から pnpm（v9.4 以上）に変更されました。
  - 公式 Docker イメージ利用時は対応不要です。

### v7.1.x を利用中の場合
参考：[v7.2.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/72x.html#管理者向け)

- [該当時] **ID/Pass 認証または SAML 認証を利用している場合**
  - 有効・無効の判定ロジックが変更されました。
  - 環境変数 `LOCAL_STRATEGY_ENABLED` / `SAML_ENABLED` を設定している場合は、アップグレード前後にプライベートブラウザ等で `/login` にアクセスし、実際の有効・無効状態が期待どおりかを確認してください。
- [該当時] **廃止された環境変数**
  - `FILE_UPLOAD_DISABLED`、`DISABLE_LINK_SHARING`が廃止されました。設定している場合は代替手段へ移行してください。
- [任意] **OpenTelemetry 設定確認**
  - v7.2.9 以降、テレメトリデータの送信がデフォルトで有効になりました。
  - 送信を望まない場合や、送信先を自組織のコレクターに変更したい場合は [テレメトリー](/ja/admin-guide/admin-cookbook/telemetry.html) を確認してください。

### v7.2.x 〜 v7.4.x を利用中の場合
参考：[v7.5.x へのアップグレード / 管理者向け](/ja/admin-guide/upgrading/75x.html#管理者向け)

- [該当時] **S3 互換オブジェクトストレージを利用している場合**
  - アップロード方式がマルチパートアップロードに変更されました。
  - IAM ポリシーに `s3:AbortMultipartUpload` 権限を追加してください（`CreateMultipartUpload`・`CompleteMultipartUpload`・`UploadPart` は `s3:PutObject` に含まれるため追加不要）。
- [該当時] **公式 Docker イメージを利用している場合**
  -  ベースイメージが Docker Hardened Images (DHI) に変更され、シェル（`sh`・`bash`）やパッケージマネージャーを含みません。
  - `docker exec` での対話的デバッグや、コンテナ内シェル操作・追加パッケージインストールを前提とした運用、独自エントリポイント・派生イメージは影響を受けるため、アップグレード前に動作を確認してください。

## 4. GROWI を最新バージョンへアップグレード
- [v7.5.x へのアップグレード](/ja/admin-guide/upgrading/75x.html)に沿って、**直接、最新バージョンへのアップグレード**を行なってください。
