# GROWI v7 系への移行ガイド

GROWI は、最新の v7 系（執筆時点の最新は v7.5.x）を利用することを推奨します。
最新系列では機能・パフォーマンス・セキュリティの改善が継続的に行われており、サポート対象の実行環境も最新化されています。

このページは、**いま運用しているバージョンを確認すれば、v7 系へ移行するために「結局なにをすればよいか」が分かる**ことを目的とした、移行情報の入口です。
まず [現在のバージョンを確認する](#現在のバージョンを確認する) で自分の系列を選び、該当する節だけを読めば、その系列の移行に必要な案内は完結します。

各マイナーバージョンの具体的な変更点・手順は、個別の「GROWI vX.Y.x へのアップグレード」ページにまとまっています。各節では、その系列で実際に確認すべき個別ガイドだけをリンクしています。すべての系列のページに目を通す必要はありません。

::: tip
最低でも v7.4.0 以降を利用することを推奨します。
Revision データマイグレーションに関する安全策が、v7.4.0 以降では自動で適用されます。
:::

## 現在のバージョンを確認する

GROWI の管理画面、または起動ログでいま動かしているバージョンを確認し、次の表から該当する系列の節へ進んでください。該当する節だけを読めば、その系列の移行に必要な案内はすべてそろっています。

| 現在のバージョン | 進む節 | 移行のポイント |
| :--- | :--- | :--- |
| 〜v4.x | [〜v4.x を利用中の場合](#〜v4-x-を利用中の場合) | v5 互換形式への変換という手作業がある。段階的に上げる |
| v5.x 〜 v6.0.x | [v5.x 〜 v6.0.x を利用中の場合](#v5-x-〜-v6-0-x-を利用中の場合) | 実行環境の要件を満たし、v7.4 以降へ直接上げる |
| v6.1.0 〜 v7.0.15 | [v6.1.0 〜 v7.0.15 を利用中の場合](#v6-1-0-〜-v7-0-15-を利用中の場合) | **情報漏洩につながる脆弱性あり。最優先で対応する** |
| v7.0.16 以降 | [v7.0.16 以降を利用中の場合](#v7-0-16-以降を利用中の場合) | 通常のアップグレード。最新系列へ上げる |


## アップグレードの基本的な考え方

GROWI は起動時にデータベースのマイグレーションを自動的に実行するため、原則として複数のバージョンを飛ばして一度にアップグレードできます。
ただし、次の 2 点だけは例外的に注意が必要です。具体的な対応は、それぞれ該当する系列の節に記載しています。

- **一度立ち止まって手作業を行うべきポイント**（特に v4.x 以前からの「v5 互換形式」への変換）
- **起動・常用も経由もしてはいけないバージョン範囲**（v6.1.0〜v7.0.15。情報漏洩と修復不能なデータ破損を引き起こす）


## 〜v4.x を利用中の場合

**経路: v5 系 → v6 系 → v7.4 以降（推奨は最新系列）**

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **v5 系へ上げる**。Elasticsearch を利用している場合は、このタイミングで **Elasticsearch 6 → 7** に対応します（[v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html)）。
3. **「v5 互換形式」への変換を手作業で実施する**。v5.0 でページの内部データ形式が大きく変わり、v4.5 以前に作成されたページは自動では変換されません。未変換のままだとページツリーへの表示や、移動・リネーム・削除時の子孫ページの扱いに影響するため、計画的に（メンテナンスモードの活用も検討して）実施してください。手順・影響範囲は [v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html) を参照してください。
4. **v6 系を通過し、v7.4 以降まで上げる**。途中で以下の実行環境・仕様変更に対応します。
   - **Node.js 16 のサポート終了（18 / 20 へ）**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **Bootstrap v4 → v5 記法の手動書き換え**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **HackMD 連携機能の廃止**（v7.0。ビルトインエディタの同時編集へ移行）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **MongoDB 6.0 以上が必須**（v7.1）— [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html)

::: danger
v6 系から上げる際は、**v6.1.0〜v7.0.15 で起動・常用しないでください。また、この範囲を経由（通過）させないでください**。この範囲には Revision（ページの更新履歴）データの重大な不具合があり、起動・常用すると非公開ページや削除済みページの内容・作成者情報が権限のないユーザーから閲覧可能になり（情報漏洩）、経由するだけでも経由時点より前の履歴データが破損して修復できなくなります。**v7.4.0 以降へ直接** 上げてください。詳細は開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を参照してください。
:::


## v5.x 〜 v6.0.x を利用中の場合

**経路: v7.4 以降へ直接（v6.1.0〜v7.0.15 を経由しない）**

v5 互換形式への変換は v5.0 で対応済みのため、この系列からは実行環境の要件を満たすことが主な作業になります。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **実行環境の要件を満たす**。
   - **Node.js 16 のサポート終了（18 / 20 へ）**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **Bootstrap v4 → v5 記法の手動書き換え**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **HackMD 連携機能の廃止**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **MongoDB 6.0 以上が必須**（v7.1）— [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html)
3. **v7.4 以降へ直接アップグレードする**。

::: danger
**v6.1.0〜v7.0.15 で起動・常用したり、この範囲を経由（通過）させたりしないでください**。この範囲には Revision データの重大な不具合があり、起動・常用すると非公開ページ・削除済みページの内容や作成者情報が権限のないユーザーから閲覧可能になり（情報漏洩）、経由するだけでも経由時点より前の履歴データが修復不能に破損します。必ず **v7.4.0 以降へ直接** アップグレードしてください。詳細は開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を参照してください。
:::

::: tip
v4.5 以前から移行してきた環境で、まだ「v5 互換形式」への変換を実施していない場合は、先に変換を済ませてください（手順は [v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html)）。v5.0 以降で新規構築した環境では不要です。
:::


## v6.1.0 〜 v7.0.15 を利用中の場合

**最優先で対応してください。** この範囲は、起動・運用しているだけで **情報漏洩につながる重大なセキュリティ脆弱性** があるバージョンです。

このバージョンでは、システム内の全 Revision（ページの更新履歴）が単一のページに誤って関連付けられ、非公開ページや削除済みページの内容・作成者・更新者・更新内容が、本来アクセス権限のないユーザーから閲覧可能になります。さらに、**この範囲を一度でも経由（アップグレードで通過）したシステムは、経由時点より前に作成された履歴データが破損し、後から修復できません**。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **直ちに v7.4.0 以降へアップグレードして漏洩を止める**。根本原因は v7.0.16 で除去されていますが、安全策が自動適用される **v7.4.0 以降** を推奨します。アップグレード時に [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html) / [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html) の実行環境要件（Node.js・Bootstrap 記法・HackMD 廃止・MongoDB 6.0 以上）も併せて満たしてください。
3. **破損状況を確認し、可能な場合は復旧する**。この範囲を経由したシステムでは、経由時点より前の履歴データは **アップグレードしても元に戻りません（修復不可）**。ごく限られた条件下でのみ手動復旧が可能です。漏洩した可能性のある情報の影響範囲・破損判定・復旧手順は、開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を確認してください。

::: danger
v6.1.0〜v7.0.15 は、新しいバージョンへ上げるまでの暫定運用であっても **絶対に起動・経由しないでください**。すでに運用してしまっている場合は、非公開ページ・削除済みページなど漏洩した可能性のある情報の影響範囲を必ず確認してください。
:::


## v7.0.16 以降を利用中の場合

**経路: 最新系列（推奨は v7.5.x）へ**

Revision データマイグレーションの根本原因は v7.0.16 で除去済みのため、通常のアップグレードとして進められます。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **個別ガイドで変更点を確認する**（[v7.1.x](/ja/admin-guide/upgrading/71x.html) 以降、通過する vX.Y.x ガイド）。各ガイドの「アップグレード前にチェックすべきこと」に目を通してください。
3. **最新系列へアップグレードする**。Revision データの安全策が自動適用される **v7.4.0 以降** を維持してください。

::: warning
過去に **v6.1.0〜v7.0.15 を経由してきた** 環境では、その経由時点より前に作成された履歴データがすでに破損している可能性があります（v7.0.16 以降へ上げても修復されません）。特に現在 v7.0.16〜v7.0.20 を利用中の場合が該当します。破損判定・復旧手順は開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を確認してください。
:::


## アップグレード前後の確認

- アップグレード前に、必ずデータベースおよび添付ファイルのバックアップを取得してください
- 各節でリンクした個別ガイドの「アップグレード前にチェックすべきこと」を確認してください
- 大規模な変換作業（v5 互換形式など）を伴う場合は、メンテナンスモードの活用を検討してください
