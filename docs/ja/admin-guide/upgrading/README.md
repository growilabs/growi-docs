# GROWI v7 系への移行ガイド

GROWI は、最新の v7 系（執筆時点の最新は v7.5.x）を利用することを推奨します。
最新系列では機能・パフォーマンス・セキュリティの改善が継続的に行われており、サポート対象の実行環境も最新化されています。

このページは、**いま運用しているバージョンを確認すれば、v7 系へ移行するために「結局なにをすればよいか」が分かる**ことを目的とした、移行情報の入口です。
まず [現在のバージョンを確認する](#現在のバージョンを確認する) で自分の系列を選び、該当する節を読んでください。

各マイナーバージョンの具体的な変更点・手順は、個別の「GROWI vX.Y.x へのアップグレード」ページにまとまっています。
アップグレードを実施する前に、通過するすべての系列の個別ガイドを必ず確認してください。

::: tip
最低でも v7.4.0 以降を利用することを推奨します。
後述の Revision データマイグレーションに関する安全策が、v7.4.0 以降では自動で適用されます。
:::

## 現在のバージョンを確認する

GROWI の管理画面、または起動ログでいま動かしているバージョンを確認し、次の表から該当する系列の節へ進んでください。
「結局なにをすればよいか」は各節の冒頭にまとめてあります。

| 現在のバージョン | 進む節 | ひとことで言うと |
| :--- | :--- | :--- |
| 〜v4.x | [〜v4.x を利用中の場合](#〜v4-x-を利用中の場合) | v5 互換形式への変換という大作業がある。段階的に上げる |
| v5.x 〜 v6.0.x | [v5.x 〜 v6.0.x を利用中の場合](#v5-x-〜-v6-0-x-を利用中の場合) | 実行環境の要件を満たし、v7.4 以降へ直接上げる |
| v6.1.0 〜 v7.0.15 | [v6.1.0 〜 v7.0.15 を利用中の場合](#v6-1-0-〜-v7-0-15-を利用中の場合) | **情報漏洩につながる脆弱性あり。最優先で対応する** |
| v7.0.16 以降 | [v7.0.16 以降を利用中の場合](#v7-0-16-以降を利用中の場合) | 通常のアップグレード。最新系列へ上げる |


## アップグレードの基本的な考え方

GROWI は起動時にデータベースのマイグレーションを自動的に実行するため、原則として複数のバージョンを飛ばして一度にアップグレードできます。
ただし、次の 2 点は例外的に注意が必要です。各節の手順は、この 2 点をどう扱うかで決まります。

- **一度立ち止まって手作業を行うべきポイント** がある（特に v4.x 以前からの「v5 互換形式」への変換）。詳細は [v5 互換形式への変換について](#v5-互換形式への変換について) を参照してください。
- **途中下車（起動・常用）してはいけないバージョン範囲** がある（v6.1.0〜v7.0.15 の Revision データマイグレーションバグ）。詳細は [避けるべき「途中下車」バージョンについて](#避けるべき「途中下車」バージョンについて) を参照してください。


## 〜v4.x を利用中の場合

**経路: v5 系 → v6 系 → v7.4 以降（推奨は最新系列）**

一連の移行の中で最も作業が大きい系列です。データ形式の変換と実行環境の更新を、段階的に進めてください。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **v5 系へ上げる**。Elasticsearch を利用している場合は、このタイミングで **Elasticsearch 6 → 7** に対応します（[v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html)）。
3. **「v5 互換形式」への変換を手作業で実施する（最大の山場）**。v5.0 でページの内部データ形式が大きく変わり、v4.5 以前に作成されたページは自動では変換されません。計画的に、メンテナンスモードの活用も検討して実施してください。詳細は [v5 互換形式への変換について](#v5-互換形式への変換について) と [v5.0.x ガイド](/ja/admin-guide/upgrading/50x.html) を参照してください。
4. **v6 系を通過し、v7.4 以降まで上げる**。途中で以下の実行環境・仕様変更に対応します。
   - **Node.js 16 のサポート終了（18 / 20 へ）**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **Bootstrap v4 → v5 記法の手動書き換え**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **HackMD 連携機能の廃止**（v7.0。ビルトインエディタの同時編集へ移行）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **MongoDB 6.0 以上が必須**（v7.1）— [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html)

::: warning
v6.1.0〜v7.0.15 のバージョンで **起動・常用したまま留まらないでください**。この範囲には Revision データマイグレーションの不具合があります。v6 系から上げる際は、この範囲で途中下車せず **v7.4.0 以降へ直接** 上げてください。理由は [避けるべき「途中下車」バージョンについて](#避けるべき「途中下車」バージョンについて) を参照してください。
:::


## v5.x 〜 v6.0.x を利用中の場合

**経路: v7.4 以降へ直接（v6.1.0〜v7.0.15 で途中下車しない）**

v5 互換形式への変換は v5.0 で対応済みのため、この系列からは実行環境の要件を満たすことが主な作業になります。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **実行環境の要件を満たす**。
   - **Node.js 16 のサポート終了（18 / 20 へ）**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **Bootstrap v4 → v5 記法の手動書き換え**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **HackMD 連携機能の廃止**（v7.0）— [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html)
   - **MongoDB 6.0 以上が必須**（v7.1）— [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html)
3. **v7.4 以降へ直接アップグレードする**。v6.1.0〜v7.0.15 の範囲で起動・常用しないでください（[避けるべき「途中下車」バージョンについて](#避けるべき「途中下車」バージョンについて)）。

::: tip
v4.5 以前から移行してきた環境で、まだ「v5 互換形式」への変換を実施していない場合は、先に変換を済ませてください（[v5 互換形式への変換について](#v5-互換形式への変換について)）。v5.0 以降で新規構築した環境では不要です。
:::


## v6.1.0 〜 v7.0.15 を利用中の場合

**最優先で対応してください。** この範囲は、起動・運用しているだけで **情報漏洩につながる重大なセキュリティ脆弱性** があるバージョンです。

このバージョンでは、システム内の全 Revision（ページの更新履歴）が単一のページに誤って関連付けられ、非公開ページや削除済みページの内容・作成者・更新者・更新内容が、本来アクセス権限のないユーザーから閲覧可能になります。さらに、**この範囲を一度でも経由（アップグレードで通過）したシステムは、経由時点より前に作成された履歴データが破損し、後から修復できません**。詳細は [避けるべき「途中下車」バージョンについて](#避けるべき「途中下車」バージョンについて) を参照してください。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **直ちに v7.4.0 以降へアップグレードして漏洩を止める**。根本原因は v7.0.16 で除去されていますが、安全策が自動適用される **v7.4.0 以降** を推奨します。アップグレード時に [v7.0.x ガイド](/ja/admin-guide/upgrading/70x.html) / [v7.1.x ガイド](/ja/admin-guide/upgrading/71x.html) の実行環境要件も併せて満たしてください。
3. **破損状況を確認し、可能な場合は復旧する**。この範囲を経由したシステムでは、経由時点より前の履歴データは**アップグレードしても元に戻りません（修復不可）**。ごく限られた条件下でのみ手動復旧が可能なケースがあります。漏洩した可能性のある情報の影響範囲・破損判定・復旧手順は、開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を確認してください。

::: danger
v6.1.0〜v7.0.15 は、新しいバージョンへ上げるまでの暫定運用であっても **絶対に経由しないでください**。すでに運用してしまっている場合は、非公開ページ・削除済みページなど漏洩した可能性のある情報の影響範囲を必ず確認してください。
:::


## v7.0.16 以降を利用中の場合

**経路: 最新系列（推奨は v7.5.x）へ**

Revision データマイグレーションの安全策が適用される範囲のため、通常のアップグレードとして進められます。

### やること

1. **バックアップを取得する**（データベースと添付ファイル）。
2. **通過する系列の個別ガイドで変更点を確認する**（[v7.1.x](/ja/admin-guide/upgrading/71x.html) 以降、各 vX.Y.x ガイド）。各ガイドの「アップグレード前にチェックすべきこと」に必ず目を通してください。
3. **最新系列へアップグレードする**。最低でも v7.4.0 以降（安全策が自動適用される範囲）を維持してください。


## 補足: 注意が必要な 2 つのポイントの詳細

ここから先は、上記の各節から参照される共通の詳細情報です。

### v5 互換形式への変換について

v4.x 以前から v5 系にアップグレードした後の **「v5 互換形式」への変換が、一連のアップグレードの中で最も大きな作業** です。
v5.0 でページの内部データ形式が大きく変わり、v4.5 以前に作成されたページは自動では変換されません。
ページツリーへの表示や、移動・リネーム・削除時の子孫ページの扱いに関わるため、計画的に実施する必要があります。

詳細な手順・影響範囲は [GROWI v5.0.x へのアップグレード](/ja/admin-guide/upgrading/50x.html) を参照してください。

### 避けるべき「途中下車」バージョンについて

GROWI v5.0.0〜v7.0.20 には Revision（ページの更新履歴）データのマイグレーション処理に関する不具合があり、特に **v6.1.0〜v7.0.15 は情報漏洩につながる重大なセキュリティ脆弱性** になっています。
このバージョンでは、本来は別々のページに紐づくべき全 Revision が単一のページに誤って関連付けられ、非公開ページや削除済みページの内容・作成者情報が、本来アクセス権限のないユーザーから閲覧可能になります。

さらに、この範囲を **一度でも経由（アップグレードで通過）したシステムは、経由時点より前に作成された履歴データが破損し、後から修復できません**。
このため、アップグレードの経路として v6.1.0〜v7.0.15 を経由しないことを強く推奨します。

修正の経緯は次のとおりです。

- **v7.0.16**: バグの根本原因を除去
- **v7.0.21**: API 側に安全弁を追加し、破損データを非表示化
- **v7.4.0 以降**: 自動判定による安全策と改善を追加（安全策が自動で適用される）

| バージョン範囲 | セキュリティリスク | データ状態 |
| :--- | :--- | :--- |
| v5.0.0〜v6.0.15 | なし | 安全（最新リビジョンのみ表示） |
| v6.1.0〜v7.0.15 | **Critical（情報漏洩）** | **破損（修復不可）** |
| v7.0.16〜v7.0.20 | 経由したバージョンによる | 経由したバージョンによる |
| v7.0.21〜v7.3.x | なし | 安全（API 側の安全弁で保護） |
| v7.4.0 以降 | なし | 安全（自動判定で適切に処理） |

※ v7.0.16〜v7.0.20 のデータ状態は、それ以前に経由したバージョンによって変わります（判定方法は開発 Wiki を参照）。

推奨される安全な経路は次のとおりです。

- v6.0.15 以前から上げる場合は、v6.1.0〜v7.0.15 を経由せず **v7.4.0 以降へ直接アップグレード** する
- 新規に構築する場合は、v7.4.0 以降（推奨は最新系列）で構築する

::: danger
この不具合の詳細（影響条件・原因・確認方法・MongoDB を用いた破損判定と復旧手順）は、開発 Wiki の [v5.0.0〜v7.0.15 での Revision データマイグレーションバグ](https://dev.growi.org/69301054963f68dfcf2b7111) を必ず確認してください。
すでに該当範囲を経由して運用している場合の対応も、こちらに記載されています。
:::


## アップグレード前後の確認

- アップグレード前に、必ずデータベースおよび添付ファイルのバックアップを取得してください
- 通過するすべての系列の個別ガイドの「アップグレード前にチェックすべきこと」を確認してください
- 大規模な変換作業（v5 互換形式など）を伴う場合は、メンテナンスモードの活用を検討してください
